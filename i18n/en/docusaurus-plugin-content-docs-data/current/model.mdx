# Data Model

Twelve tables linked by foreign keys. The core is the gazetteer and the
itinerary; everything else references them and the source registry.

| Table | What it holds | Rows |
|---|---|---|
| `sources.csv` | The source registry with a `rights` column; every `source_id` in other tables is a foreign key into this table | 9 |
| `places.csv` | A gazetteer of **unique** places (Bidar, Hormuz, Lar — each once), with coordinates and authority IDs | 26 |
| `itinerary.csv` | The **ordered** 29 legs of the route; `place_id` is a key into `places` (repeats allowed) | 29 |
| `legs.csv` | 28 transitions between adjacent points: distances (generated by `tools/build_legs.py`) | 28 |
| `events.csv` | Key events along the route: place, participants, type, dating | 12 |
| `people.csv` | Participants: who helped, who harmed, who is mentioned; cross-checked against VIAF/Wikidata | 15 |
| `edges.csv` | Edges of the people network (helped / harmed / neutral / indirect) | 17 |
| `fragments.csv` | 104 text fragments (Fol. 1–104): layer, genre, period, place. **Metadata only** — the `quote` field is empty (rights-restricted) | 104 |
| `calendar.csv` | Computed Easters, Great Lents, and Ramadans for 1467–1475 (generated by `tools/computus.py`) | 37 |
| `editions.csv` | Publication milestones of the text, 1818–2026 | 9 |
| `citations.csv` | Estimated scholarly attention by decade (`epistemic=model` — a heuristic) | 12 |
| `trade.csv` | Trade information: 4 textual facts + 19 modeled entries | 23 |

## Conventions

- **Encoding**: UTF-8 without BOM; CSV per RFC 4180.
- **`epistemic`** — a five-value vocabulary for the kind of claim:
  `text` (stated in the Journey) · `reconstruction` (chronology
  reconstruction) · `localization` (map anchoring) · `model` (heuristic
  estimate) · `hypothesis` (a hypothesis).
- **`certainty`** — a separate reliability axis: `certain` · `approx` · `disputed`.
- **`recon_status`** for places and people: `confirmed` / `candidate` / `none` /
  `collective` / `skip-region`; authority IDs are recorded **only** for
  `confirmed` rows — a wrong QID is worse than an empty one.
